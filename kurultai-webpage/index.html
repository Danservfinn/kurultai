How It Works

A seamless delegation flow that ensures your requests reach the right expertise.

1

You Send a Request

Your message arrives at Kublai, the main agent. Kublai reads personal context from files and queries operational knowledge from Neo4j.

2

Kublai Creates a Task

Kublai creates a Task node in Neo4j with status "pending" and identifies which specialist agent is best suited for the work.

3

Delegation via Agent-to-Agent Messaging

Kublai sends a message directly to the specialist agent using OpenClaw's native agentToAgent messaging capability.

4

Specialist Claims & Completes

The specialist claims the task (status: "in_progress"), completes the work, stores results in Neo4j, and notifies Kublai.

5

Kublai Synthesizes & Responds

Kublai marks the task complete, synthesizes the specialist's work with personal context, and delivers the final response to you.
