- Vault has valid Z.AI credentials (b5b1f9537dda43198c640b9a3b48f79b...)
- `~/.claude/credentials.json` does NOT exist (OAuth not logged in)
- 10+ `*.failed.done.md` files in temujin/tasks/

**Code location:** `/Users/kublai/.openclaw/agents/main/scripts/agent-task-handler.py:2087-2094`

## Why CIRCULAR_DEPENDENCY_GUARD Didn't Fire

## Fleet-Wide Impact

**ALL 6 agents** show `is_using_proxy=False`. Any agent without ANTHROPIC_* variables in settings.json will fail the same way. This is a systemic bug.

## Required Actions

### Immediate (Human Intervention Required)
1. Run `claude auth login` for temujin's claude-code session, OR
2. Update agent-task-handler.py with vault variable mapping (see code below)

### Fix Code (agent-task-handler.py:2087-2094)
```python
# Map vault variables to Claude Code format
VAULT_VAR_MAP = {
    'ZAI_AUTH_TOKEN': 'ANTHROPIC_AUTH_TOKEN',
    'ZAI_BASE_URL': 'ANTHROPIC_BASE_URL',
    'ZAI_MODEL': 'ANTHROPIC_MODEL',
    'ALIBABA_AUTH_TOKEN': 'ANTHROPIC_AUTH_TOKEN',
    'ALIBABA_BASE_URL': 'ANTHROPIC_BASE_URL',
    'ALIBABA_MODEL': 'ANTHROPIC_MODEL',
}

vault_creds = load_vault_credentials()
for vault_key, value in vault_creds.items():
    if vault_key in VAULT_VAR_MAP:
        env[VAULT_VAR_MAP[vault_key]] = value
    else:
        env[vault_key] = value
```
