# TICK Log Fallback for Stale Tock Data

**Date:** 2026-03-12
**Agent:** temujin
**Category:** heartbeat_system

## Problem

Reflection context generated by `kurultai_brainstorm.py` was showing stale system health data (Redis/Neo4j status as "unknown") because:
1. Tock data is collected every 30 minutes
2. Reflection runs at various times, sometimes before fresh tock data is available
3. No fallback mechanism existed to get current system status

## Solution

Added TICK log fallback mechanism in `kurultai_brainstorm.py`:

1. **New function `_parse_tick_log_for_system_health()`** - Parses the latest TICK log line directly to extract Neo4j and Redis status

2. **Modified system_health section** - Checks tock file age before using:
   - If tock is fresh (< 10 minutes old): use tock data (normal path)
   - If tock is stale (> 10 minutes): fall back to TICK log parsing

3. **TICK runs every 5 minutes** via launchd, providing more frequent status updates than tock (30 minutes)
