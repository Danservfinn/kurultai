# Proposal: Fix auth preflight caching to reduce concurrent API calls

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-12 06:33:23
**Domain:** heartbeat_system
**Model:** claude-opus-4-6

## Problem

Multiple agents doing simultaneous auth checks cause resource contention and timeouts. The auth_health_preflight() function in both agent-task-handler.py and task-watcher.py always ran fresh checks, leading to "simultaneous_failures" patterns (3+ agents failing in same minute).

## Solution

Modified auth_health_preflight() in both scripts to use cached auth-heartbeat.json results. Fresh auth checks now only run when cache is stale (>5 minutes) or missing.
