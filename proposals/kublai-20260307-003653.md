# Proposal: Prevent zombie task accumulation via hard execution ceiling and disk-aware dispatch

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-07 00:36:53
**Domain:** task_dispatch
**Model:** opus

## Problem

Task-watcher lost in-memory tracking after restart, dispatching 2nd tasks per agent alongside 6h+ zombie processes. recover_stale_executions() skipped recovery when PID was alive, causing indefinite zombie accumulation and queue stalls.

## Solution

task-watcher.py: (1) Added HARD_MAX_EXECUTING_AGE=7200s that kills processes regardless of PID status after 2h; (2) watch_cycle() now checks for .executing files on disk before dispatching, preventing double-dispatch after restart.
