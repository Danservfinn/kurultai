# Proposal: Fix Neo4j reconciliation misclassifying failed tasks as completed

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-07 08:33:06
**Domain:** state_management
**Model:** opus

## Problem

sync_check() in neo4j_task_tracker.py used `if '.done' in fname` to detect completed tasks, which also matched `.failed.done.md` and `.orphan-failed.done.md` files. All failed tasks were reconciled to COMPLETED in Neo4j, corrupting capability scores, routing decisions, and health metrics system-wide.

## Solution

Changed file status detection in neo4j_task_tracker.py:sync_check() (line 508-522) to check `.failed.done` and `.orphan-failed.done` before the generic `.done` pattern, correctly returning FAILED status for failed tasks.
