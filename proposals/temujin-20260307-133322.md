# Proposal: Fix keyword routing to use word-boundary matching instead of substring matching

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-07 13:33:22
**Domain:** routing_pipeline
**Model:** opus

## Problem

route_by_text and 5 other functions used plain substring matching (kw in text_lower) causing false positives — "ui" matched inside "build"/"guide"/"require", "api" matched inside "capital", "fix" matched inside "prefix". This inflated temujin's scores and caused sticky routing (e.g., "Write a guide for building signup" routed to temujin instead of chagatai).

## Solution

Added _kw_match() helper with word-boundary regex for single-word keywords, applied it to route_by_text, detect_skill_hint, get_agent_scores, get_capable_alternates, can_handle_task, and misroute detection scoring — all in task_intake.py
