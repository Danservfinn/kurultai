# Proposal: Fix KeyError crash in rule_registry.py that silently blocked all memory maintenance

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-12 22:36:52
**Domain:** memory_architecture
**Model:** claude-opus-4-6

## Problem

`memory_audit.py --fix` was crashing with `KeyError: 'text'` whenever rule entries in `rules.json` lacked the "text" field. This prevented ALL memory maintenance: 96 rules couldn't sync, 32 stale log files (252+ KB) weren't cleaned, context compaction was blocked, and duplicate/dead rules accumulated. The crash was silent from cron — no fix was ever applied.

## Solution

Changed `r["text"]` to `r.get("text", "")` in two locations in `scripts/rule_registry.py` — `add_rule()` (line 139) and `_extract_and_add_rules()` (line 281). After fix, ran `memory_audit.py --fix` which synced 96 rules across 6 agents, deleted 32 stale log files, and now exits clean (code 0, no criticals).
