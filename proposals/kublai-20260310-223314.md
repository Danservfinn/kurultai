## Problem

kublai-actions.py was hardcoding all cron fix tasks to temujin, bypassing the scoring system that correctly identifies ogedei for ops/infrastructure tasks. This caused tasks like "review parse monetization blockers" to route to temujin hourly despite never completing, when ogedei was the optimal agent.

## Solution

Changed create_task call for cron fixes from agent="temujin" to agent=None (auto-route), added VALID_AGENTS import, and updated pending task check to scan all agents
