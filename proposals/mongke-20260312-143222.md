## Problem

Neo4j connection failures cause mongke's research tasks to fail (0% first-attempt success rate). Scripts directly use `GraphDatabase.driver()` without exception handling, causing crashes when Neo4j is unavailable.

## Solution

Added `safe_neo4j_op()`, `execute_query_cypher()`, and `check_neo4j_available()` functions to `scripts/neo4j_utils.py` that catch Neo4j exceptions and return fallback values instead of crashing. Also added M005 behavioral rule documenting the pattern.
