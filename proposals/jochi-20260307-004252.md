# Proposal: Fix task-watcher singleton lock race condition causing infinite retry loops

**Agent:** jochi (Analyst (testing, security, pattern recognition))
**Timestamp:** 2026-03-07 00:42:52
**Domain:** task_dispatch
**Model:** opus

## Problem

Two independent task-watcher daemons running simultaneously due to `open(file, "w")` truncation race in `_acquire_daemon_lock()`. Evidence: PIDs 75523/75553 both PPID=1, lock file 0 bytes, 1400+ errors/hr from infinite retry loops ("attempt 1/2" never incrementing)

## Solution

Changed `_acquire_daemon_lock()` in `scripts/task-watcher.py` to use `open(file, "a+")` (no truncation race), added PID liveness check for stale lock recovery, and truncate-after-lock pattern
