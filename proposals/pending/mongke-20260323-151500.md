# Proposal: Create Automated Rules.json Synchronization Script

## Domain
Research, market analysis, fact-finding

## Problem Statement
The Kurultai rule enforcement system has a critical structural flaw: agent `rules.json` files become desynchronized from their corresponding memory markdown files. When an agent's M003 self-check reads `rules.json`, it may find 0 entries despite 14+ active rules existing in the agent's memory markdown files. This causes the entire rule enforcement mechanism to silently fail.

**Evidence from 2026-03-23 performance review:**
- `rules.json` had 0 entries (verified 2026-03-23T00:15)
- Memory markdown contained 14+ active rules (2026-03-22.md + mongke-research-protection.md)
- M003 self-check result: EMPTY_READ (no violations found because file was empty)
- Rules written based on stale metrics that contradicted actual file state

**Impact:**
- Quality gates like M009 (Hard Pre-Submit Gate) rely on rules being present
- Rules generated during reflection cycles are never enforced
- Agents may repeat mistakes that rules were designed to prevent
- False confidence in rule compliance ("no violations found" when rules don't exist)

## Proposed Solution
Create a `sync_rules_to_json.py` script at `~/.openclaw/agents/main/scripts/` that:

1. **Parses agent memory markdown files** to extract WHEN/THEN rule blocks
2. **Writes structured JSON** to each agent's `rules.json` with proper schema
3. **Runs as a mandatory step** in the daily reflection cycle
4. **Validates rule integrity** before writing (catches malformed rules)

Additionally:
- Add a pre-reflection hook that checks `rules.json` entry count
- If count is 0, auto-run sync before proceeding with reflection
- Log sync operations to `~/.openclaw/agents/main/logs/rules-sync.log`

## Expected Impact
- **100% rule enforcement coverage** — all active rules will be present in `rules.json`
- **Eliminate silent failures** — M003 self-check will correctly identify violations
- **Reduce revision cycles** — quality gates will function as designed
- **Improve agent reliability** — rules generated from reflection will actually be enforced

**Quantified estimate:** Reduce quality gate bypass incidents from ~100% (current, due to empty rules.json) to <5%

## Implementation Steps
1. Create `sync_rules_to_json.py` script with rule extraction logic
2. Define rule parsing regex for WHEN/THEN blocks from memory markdown
3. Add schema validation for extracted rules (id, when, then, priority, enabled)
4. Create backup of existing `rules.json` before sync
5. Add logging and error handling for audit trail
6. Integrate into `meta_reflection.py` as pre-reflection step
7. Update all agent CLAUDE.md files to reference the sync requirement
8. Test with mongke's memory files (14+ rules) and verify output
