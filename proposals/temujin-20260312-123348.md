# Proposal: Fix circuit breaker health script Python 3.9 compatibility

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-12 12:33:48
**Domain:** state_management
**Model:** claude-opus-4-6

## Problem

The circuit breaker health script was crashing every 5 minutes with `TypeError: unsupported operand type(s) for |: 'type' and 'type'` because cron was using Python 3.9.6 (`/usr/bin/python3`) which doesn't support the `str | Path | None` union syntax. This prevented OPEN agents from auto-transitioning to HALF_OPEN, causing circuit breaker deadlock.

## Solution

Added `from __future__ import annotations` to 7 scripts (circuit_breaker.py, circuit_breaker_health.py, evaluate_agent_rules.py, kurultai-monitor.py, quality_gate.py, json_registry.py, retry_utils.py, self_healing_score.py) to enable PEP 604 union syntax for Python < 3.10 compatibility. Also updated cron job to use explicit `/usr/bin/python3` path.
