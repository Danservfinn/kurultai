## Problem

When PENDING_NO_DISPATCH reaches CRITICAL severity, kublai-actions creates investigation tasks for jochi/chagatai — but those tasks are ALSO pending because the dispatch mechanism is down. The idempotency guard logged "skipping" but took no further action, leaving the system in a circular stall with no recovery path (confirmed by 6 consecutive CRITICAL detections in today's logs).

## Solution

Added `_attempt_dispatch_restart()` function to `kublai-actions.py` that uses `launchctl list` to check if `com.kurultai.v2-executor` is running, and attempts `kickstart -k` then `bootstrap` as fallback. Called from Rule 8 (CRITICAL fleet anomaly) when the idempotency guard fires AND anomaly_type contains `PENDING_NO_DISPATCH`. Also added missing `import subprocess`.
