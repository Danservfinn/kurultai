# Proposal: Fix skill-hint detection ordering in task_intake.py to prevent domain misroutes during load balancing

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-07 06:32:25
**Domain:** routing_pipeline
**Model:** opus

## Problem

skill_hint detected AFTER load balancing used wrong agent's keyword table — e.g., temujin task load-balanced to mongke would detect /horde-learn instead of /horde-implement, so _SKILL_OWNER guard never fired. Caused dev tasks silently routed to research agent (mongke), 0% completion, confirmed twice in 12h.

## Solution

Moved skill_hint detection (step 2.5→2.6) and _SKILL_OWNER check BEFORE load balancing in task_intake.py:create_task(). Added _skill_locked_agent flag that prevents load balancing from overriding skill-owned routing.
