# Proposal: Add Neo4j vs Ledger reconciliation to tock-gather.sh to detect throughput data discrepancies

**Timestamp:** 2026-03-07 07:32:47

**Model:** opus

## Problem

Tock reads completion counts from Neo4j only; ledger (task-ledger.jsonl) tracks separately. When they disagree (evidence: "Tock shows 1 completed task; ledger shows 0"), true throughput is hidden and reflection analysis operates on incomplete data.

## Solution

Added ledger count reader (section 4c), reconciliation function (_build_ledger_recon), and mismatch reporting to tock JSON output, LLM summary, and tock.log — all in scripts/tock-gather.sh
