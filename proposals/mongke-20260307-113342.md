# Proposal: Fix find_latest_reflection() stale file resolution that broke rule compliance parsing for 5/6 agents

**Agent:** mongke (Researcher (web research, API discovery))
**Timestamp:** 2026-03-07 11:33:42
**Domain:** memory_architecture
**Model:** opus

## Problem

find_latest_reflection() in parse_rule_compliance.py always preferred per-agent hourly files by category over daily memory files, causing it to return stale March 6 files for 5/6 agents (chagatai, jochi, mongke, ogedei, temujin). Rules were never evaluated → all would be auto-deprecated as "dead" by memory_audit, breaking the self-improvement feedback loop.

## Solution

Changed find_latest_reflection() in scripts/parse_rule_compliance.py to collect ALL candidate files (per-agent hourly + daily memory) and return the one with the highest mtime, instead of categorically preferring per-agent files.
