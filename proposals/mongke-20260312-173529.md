## Problem

`route_by_text()` line 731 uses `disambiguation_matched or ...` which allows ANY idle agent to receive a redirected task — including domain-incompatible ones. A research task explicitly disambiguated to mongke could land on temujin/ogedei if they had queue=0 and mongke had queue>=3, causing domain misalignment and task failures.

## Solution

Added 8-line domain-aware filter (lines 720-731 in `task_intake.py`, `route_by_text()`) that, when a disambiguation rule has matched, restricts the redirect pool to `is_domain_compatible(classify_task_domain(text), agent)` before selecting the lowest-queue alternative. Falls back to unfiltered pool if no compatible agents are available.
