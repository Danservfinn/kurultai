## Problem

`mark_task_completed()` (agent-task-handler) and `recover_stale_executions()` (task-watcher) both try to rename the same `.executing` file. When handler won race but recovery's rename failed with bare `except OSError: continue`, the task state was lost, causing "VERIFY FAIL" and inflating queue depth metrics (mongke showing 5 queued when actual was 2-3).

## Solution

Added race detection in 7 locations across 3 recovery phases (DEDUP, DEAD_PID, stale recovery). After OSError during rename, check `_already_completed()` to detect if handler won race, log appropriately, and continue cleanly instead of losing task state.
