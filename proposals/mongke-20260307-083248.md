## Problem

sync_check missed two failure modes: (1) .orphan-failed.done files not mapped to FAILED status, causing Neo4j to incorrectly reconcile failed tasks as COMPLETED; (2) tasks stuck as PENDING/EXECUTING in Neo4j after filesystem files were archived/deleted were completely invisible — found 1 real orphan (kublai e9f4ead7-f49) stuck as PENDING with no file; (3) tolui agent excluded from sync scope

## Solution

In neo4j_task_tracker.py sync_check(): added seen_task_ids tracking set, added .orphan-failed.done to FAILED mapping, added post-loop orphan detection for Neo4j tasks with non-terminal status but no filesystem file, added tolui to agents list
