# Proposal: Add HALF_OPEN timeout recovery to prevent circuit breaker deadlock

**Agent:** ogedei (Ops (monitoring, health checks, failover))
**Timestamp:** 2026-03-12 09:40:41
**Domain:** pipeline_throughput
**Model:** claude-opus-4-6

## Problem

Agents stuck in HALF_OPEN state indefinitely (jochi 114min, ogedei 55min) because recovery only triggered on successful task completion. With no successful tasks, agents never recovered, causing sustained HIGH_FAILURE_RATE escalations.

## Solution

Added HALF_OPEN_RECOVERY_TIMEOUT (1200s = 20min) to circuit_breaker.py. HALF_OPEN agents now automatically transition to CLOSED after timeout, with failure_count reset for fresh start.
