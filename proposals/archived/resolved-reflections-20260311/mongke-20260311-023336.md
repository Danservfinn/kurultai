## Problem

Local LLM (Qwen3.5-9B) used in heartbeat TICK_LLM triage was hallucinating technical issues not present in tick data (e.g., claiming "Neo4j disconnected" when tick summary showed `neo4j=up`), causing false-positive escalation to kublai

## Solution

Added DATA-VERIFICATION FILTER in watchdog-gather.sh (lines 911-953) that cross-checks LLM assertions against actual tick data before accepting alerts; filters Neo4j, Redis, and gateway "down" claims when data shows "up"
