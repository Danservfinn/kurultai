# Proposal: Fix stall-detector escalation loop with task-specific cooldown tracking

**Domain:** task_dispatch
**Model:** claude-opus-4-6

## Problem

The escalation loop occurred because stall-detector used agent-level cooldown (`task_stall:temujin`) but checked for generic pending titles "Investigate stalled task:" - different stalled tasks for the same agent bypassed both checks, creating duplicate investigations every 5 minutes. This caused jochi's queue to fill with identical investigations and zero throughput.

## Solution

Modified `kublai-actions.py` stall detection to: (1) Extract task filename from STALL_WARNING output, (2) Use task-specific cooldown key `task_stall:{agent}:{filename}`, (3) Include filename in pending task check. Each specific stalled task now gets its own 1-hour cooldown instead of agent-level throttling.
