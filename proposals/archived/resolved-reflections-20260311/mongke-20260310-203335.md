# Proposal: Fix mongke self-tasking Neo4j driver closure bug

**Domain:** reflection_pipeline
**Model:** claude-opus-4-6

## Problem

mongke_self_task.py was calling `driver.close()` on the singleton driver, breaking Neo4j connections for all subsequent queries. This caused "Driver closed" errors and prevented mongke from discovering stale knowledge and generating research tasks. Evidence: routing validation passed (rules work), but self-task queries failed with "Driver closed".

## Solution

Removed `driver.close()` calls from `_find_stale_knowledge()` and `_find_research_needed_nodes()` functions. The singleton driver is managed by `neo4j_task_tracker.py` and should only be closed at process exit via the atexit handler. Changed from `try:/finally: driver.close()` to simple `with driver.session():` context manager usage.
