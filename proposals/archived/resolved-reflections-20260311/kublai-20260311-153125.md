# Proposal: Fix tick parsing bug in kublai-actions.py

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-11 15:31:25
**Domain:** state_management
**Model:** claude-opus-4-6

## Problem

kublai-actions.py read 10th-from-last tick instead of most recent, causing false "3013s old" warnings and stale throughput decisions. Root cause: `range(min(len(lines), 10), 0, -1)` starts at i=10 → reads `lines[-10]` first.

## Solution

Changed loop to `range(1, min(len(lines), 10) + 1)` to read `lines[-1]` first (most recent tick). Also added `tests/test_tick_parsing.py` to prevent regressions.
