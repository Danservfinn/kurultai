## Problem

Cron fix tasks titled "Fix cron job: {name}" were misclassified as "implementation" domain (due to "fix" keyword matching before "cron"), routing them to temujin instead of ogedei. This caused 45% missed opportunity rate when idle agents were bypassed.

## Solution

Added `skill_hint="/kurultai-health"` parameter to cron fix task creation in `kublai-actions.py` line 435. Skill hints take precedence over keyword classification, forcing "ops" domain and routing to ogedei.
