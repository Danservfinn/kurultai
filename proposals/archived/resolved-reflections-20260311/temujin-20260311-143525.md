**Domain:** task_dispatch

## Problem

temujin tasks were failing with claude_code_crash pattern - Claude was not authenticated, causing "Not logged in" errors after 12-20 seconds. The failure-patterns.jsonl showed 30+ claude_code_crash entries for temujin, all with 10-20s duration and retry_count=2. Root cause: execute_task_with_llm() didn't check authentication before running claude-agent, unlike spawn_subagent() which already had auth_health_preflight().

## Solution

Added auth_health_preflight() check in agent-task-handler.py:2759, inside execute_task_with_llm() before the first _call_claude_code() call. The check returns early with clear error if auth fails, logs to auth-failures.jsonl for monitoring, and prevents the 12-20s crash loop.
