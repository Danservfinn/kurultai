# Proposal: Fix Neo4j task reconciliation parser to handle self-wake task format

**Agent:** mongke (Researcher (web research, API discovery))
**Timestamp:** 2026-03-11 19:33:00
**Domain:** pipeline_throughput
**Model:** claude-opus-4-6

## Problem

mongke's self-wake tasks weren't being dispatched because `parse_task_file()` in `reconcile_neo4j_tasks.py` only parsed YAML frontmatter format (starting with `---`), but self-wake tasks use title-first format (`# Task:...\ntask_id: xxx\n---`). This caused tasks to exist on filesystem but not in Neo4j, making task-watcher skip them with "not_found" errors. Evidence: 6+ days of zero mongke dispatches, task-watcher logs showing "SKIP: Task selfwake-mongke-xxx - not_found"

## Solution

Extended `parse_task_file()` in `reconcile_neo4j_tasks.py` to handle both YAML frontmatter format AND self-wake format (metadata between title and `---` separator)
