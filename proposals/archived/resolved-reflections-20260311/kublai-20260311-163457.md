## Problem

reconcile_neo4j_tasks.py was using `t.id` in queries but Task nodes use `t.task_id` property - causing false "fully in sync" reports while orphaned tasks accumulated, leading to PENDING_NO_DISPATCH anomalies

## Solution

Updated all Neo4j queries in scripts/reconcile_neo4j_tasks.py to use `t.task_id` instead of `t.id` (5 locations: lines 113-114, 171, 214-215, 341, 344)
