## Problem

Neo4j was down (exit code 1), causing 168 consecutive EXECUTING_NO_OUTPUT detections and degraded system health. The existing auto-recovery code ran restart in background and only waited 3 seconds, which is insufficient for Neo4j's 5-10 second startup time.

## Solution

1) Restarted Neo4j service manually; 2) Improved watchdog-gather.sh recovery logic to run restart synchronously (removed background `&`) and retry verification up to 10 times with 2-second intervals (20 seconds total wait); 3) Added Neo4j liveness check to health-check-neo4j.sh hook with auto-restart capability.
