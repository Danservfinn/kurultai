# Proposal: Queue-aware tiebreaking in route_by_text() to prevent sticky routing to temujin

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-11 06:34:33
**Domain:** routing_pipeline
**Model:** claude-opus-4-6

## Problem

route_by_text() defaulted to "temujin", 0 and didn't break keyword score ties by queue depth, causing biased routing when multiple agents matched equally (e.g., "deploy api" matched temujin=2, ogedei=2 but always routed to temujin)

## Solution

Modified route_by_text() in task_intake.py to: (1) collect all agent scores, (2) find tied agents with best score, (3) break ties by selecting agent with lowest queue depth, (4) for no-keyword matches, select lowest queue agent
