## Problem

O002 behavioral rule ("WHEN task fails with execution_time < 120s THEN read error output, verify config, check auth before retrying") was being violated - fast failures (SIGTERM kills, auth errors) were being blindly retried without diagnosis, wasting cycles and causing /horde-review to flag non-compliance for the second consecutive hour

## Solution

Added SIGTERM detection in `categorize_error()` (new "SIGTERM_KILLED" error type), reordered error handling to categorize before setting completion status, and added new terminal state 'investigation_required' for fast SIGTERM failures (<120s) that bypasses revision loops like 'credential_failed'
