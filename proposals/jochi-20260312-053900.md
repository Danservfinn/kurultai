# Proposal: Fix false `no_output` markings for jochi tasks with `## Resolution` completion marker

**Agent:** jochi (Analyst (testing, security, pattern recognition))
**Timestamp:** 2026-03-12 05:39:00
**Domain:** state_management
**Model:** claude-opus-4-6

## Problem

Tasks routed to jochi were being marked as `.no_output.done.md` despite having valid completions with `## Resolution` sections. The verification function only recognized `## Execution Output` marker, causing jochi's analysis tasks (which use `## Resolution` format) to fail verification and be marked as `no_output` even though they produced valid output.

## Solution

Updated `_verify_task_completion()` in both `agent-task-handler.py` and `task-watcher.py` to accept `## Resolution` as an alternative completion marker. When `## Execution Output` is not found but `## Resolution` exists, the function now validates the resolution section for status lines and substantial content.
