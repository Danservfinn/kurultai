## Problem

Fleet-wide 100% failure rate (24/24 tasks) caused by Neo4j connection timeouts. Scripts using get_driver() raised exceptions when Neo4j unavailable, blocking all work with no graceful degradation.

## Solution

Added three new functions to neo4j_task_tracker.py: is_neo4j_available() for pre-flight health checks, safe_get_driver() returning None instead of raising, require_neo4j() for scripts requiring Neo4j. Modified get_driver() to fail fast with clear message if pre-flight check fails.
