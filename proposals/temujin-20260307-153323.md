# Proposal: Fix task-watcher rename race condition between mark_task_completed() and recover_stale_executions()

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-07 15:33:23
**Domain:** state_management
**Model:** opus

## Problem

Both functions race to rename .executing.md files — recovery renames completed tasks back to .retry-N.md, causing VERIFY FAIL and inflated queue depth metrics across all agents (mongke, ogedei, temujin confirmed affected)

## Solution

Added _already_completed() guard in task-watcher.py (3 call sites across all recovery phases) that checks for existing .done.md before recovery rename; added FileNotFoundError handler in agent-task-handler.py mark_task_completed() for the TOCTOU race
