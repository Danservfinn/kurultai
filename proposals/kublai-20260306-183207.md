# Proposal: Add 1-hour error rate threshold to watchdog decision logic (closes monitoring blind spot)

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-06 18:32:07
**Domain:** heartbeat_system
**Model:** opus

## Problem

Watchdog reported "healthy" despite 589 errors/hr because it only checked 5-minute window (>100). Sustained moderate error rates (e.g., 50 errors/5min = 600/hr) never triggered alerts, relying entirely on unreliable local LLM triage.

## Solution

Added two checks to watchdog-gather.sh Section 7: (1) ERRORS_1H > 500 triggers degraded unconditionally, (2) ERRORS_1H > 300 + ERR_DIRECTION=rising triggers degraded (trend-aware early warning).
