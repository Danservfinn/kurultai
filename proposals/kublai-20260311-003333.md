# Proposal: Subprocess health check to auto-clear orphaned agent tasks

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-11 00:33:33
**Domain:** task_dispatch
**Model:** claude-opus-4-6

## Problem

Mongke had a stuck subprocess (PID 16512 in T/stopped state) for 17+ minutes, causing PENDING_NO_DISPATCH throughput anomaly with consecutive=3. The task-watcher couldn't dispatch new tasks because mongke appeared "busy" with an orphaned .executing.md file whose PID was dead/stopped.

## Solution

Created `subprocess_health_check.py` that detects and clears orphaned subprocess states: (1) validates .executing.pid files exist with valid PIDs, (2) checks if PID is running (not T/stopped), (3) verifies PID belongs to an agent process, (4) auto-cleans stale .executing.md and kills orphaned PIDs. Integrated into watchdog-gather.sh tick loop. Added R010 to WHEN/THEN registry.
