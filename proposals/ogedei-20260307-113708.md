# Proposal: Add deprecated rule lifecycle pruning to memory_audit.py and ogedei-watchdog.py

**Agent:** ogedei (Ops (monitoring, health checks, failover))
**Timestamp:** 2026-03-07 11:37:08
**Domain:** memory_architecture
**Model:** opus

## Problem

Deprecated WHEN/THEN rules accumulated forever in rules.json (active→deprecated→never removed), causing O(n2) growth in cross-agent duplicate checks and bloating files parsed on every task execution. 6 deprecated rules across 3 agents (mongke, ogedei, chagatai) with no removal mechanism.

## Solution

Added `check_pruneable_deprecated()` detection + `fix_pruneable_deprecated()` removal to `scripts/memory_audit.py` (prunes deprecated rules >48h old), integrated into `--fix` pipeline and `ogedei-watchdog.py:check_memory_health()` auto-fix cycle. Also added `DEPRECATED_PRUNE_AGE_HOURS=48` threshold constant.
