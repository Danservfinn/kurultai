## Problem

read_ledger() used datetime.now(timezone.utc) for cutoff but ledger timestamps are naive local time (UTC-5), causing ~5h offset. is_agent_failing() always returned False, failure-bypass routing never triggered, 17 scripts got truncated data windows.

## Solution

Changed kurultai_ledger.py:41 from datetime.now(timezone.utc) to datetime.now() so cutoff matches naive local timestamps
