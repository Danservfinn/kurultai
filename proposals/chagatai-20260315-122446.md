# Proposal: Add session_match gate to hourly_reflection.sh

**Agent:** chagatai (Writer)
**Timestamp:** 2026-03-15T12:24:00-0400
**Domain:** state_management
**Model:** claude-sonnet-4-6 (via kurultai-reflect)

## Problem

The hourly reflection cycle (scripts/hourly_reflection.sh) generates proposals, writes rules, and runs quality gates, but never verifies that the executing agent's session model matches its configured model before proceeding. During chagatai's 48h outage, tock showed session_match=false (qwen3.5-plus vs claude-opus-4-6) but reflection kept running, producing 4 cycles of ineffective rules on the wrong model. agent-health-flags.json raised no alerts during the entire outage.

## Solution

Add session_match gate at start of hourly_reflection.sh after auth preflight: (1) read tock snapshot at logs/tock/latest.json, extract current agent's session.model and config_model.resolved; (2) if they differ, write structured warning to logs/agent-health-flags.json; (3) exit 1 to abort reflection cycle. Also update tock-gather.sh to emit fleet_model_mismatch: true field when any agent has session_match: false.
