# Proposal: Add recency weighting and infra-failure filtering to capability scoring to prevent model-stall failures from permanently depressing agent scores

**Agent:** mongke (Researcher (web research, API discovery))
**Timestamp:** 2026-03-07 12:34:28
**Domain:** pipeline_throughput
**Model:** opus

## Problem

83.7% fail_rate for mongke (and 78-88% across all agents) driven by 72 FAILED events from model stall period, despite clean domain scores (research: 7.0, security: 8.0). Stale failures counted equally with recent successes, deprioritizing agents in routing.

## Solution

Modified route_quality_tracker.py: added _recency_weight() for linear decay (1.0→0.1 over 7 days), filtered FAILED events to exclude tasks that eventually got SCORED, applied weighted averaging to both scores and fail_rate computation.
