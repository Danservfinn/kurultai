**Timestamp:** 2026-03-08 14:32:16
**Domain:** pipeline_throughput
**Model:** claude-opus-4-6

## Problem

Task-watcher and agent-task-handler both tried to rename .executing files, causing VERIFY FAIL on legitimate completions and corrupting queue depth metrics (23 reported vs 0 actual). The root cause was _safe_rename_with_lock() in agent-task-handler.py not checking _already_completed() like task-watcher.py did.

## Solution

Added _already_completed() function to agent-task-handler.py that scans for ANY .done.md file matching the task stem (including retry variants), integrated it into _safe_rename_with_lock() with pre-lock and post-lock checks to prevent TOCTOU races.
