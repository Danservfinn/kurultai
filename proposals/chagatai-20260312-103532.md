## Problem

Task "Create documentation for recently completed features" was routed to chagatai but REJECTED by title_domain_conflict_reject because generic keywords "create"+"features" scored higher for "implementation" (score 2) than "documentation" (score 1), even though the skill_hint `/content-research-writer` correctly classified the domain as documentation.

## Solution

Modified task_intake.py lines 3072-3119 to add a score threshold check when domain came from skill hint: (1) If classified domain has 0 keyword matches but conflict domain has >0, reject (skill hint clearly wrong), (2) Otherwise only reject if conflict score is >2x classified score (weak conflicts allowed through).
