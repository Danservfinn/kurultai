# Proposal: Fix Neo4j orphan detection in sync_check to query ALL non-terminal tasks regardless of age

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-07 08:34:42
**Domain:** state_management
**Model:** opus

## Problem

sync_check() only queried Neo4j tasks from last 24h (P1D), so non-terminal tasks older than 24h accumulated as phantom PENDING/EXECUTING entries forever. Found 15 orphaned tasks (11-13h+ old) polluting Neo4j metrics — reflections and hourly reports showed inflated pending/executing counts, leading to incorrect routing decisions.

## Solution

Added separate Neo4j query in sync_check() for ALL non-terminal tasks (status IN PENDING/ASSIGNED/EXECUTING/running/ready) with no time filter. Forward pass still uses P1D for recent status mismatches. Reverse pass now uses unbounded non-terminal query to catch orphans at any age. File: neo4j_task_tracker.py:sync_check() lines 462-556.
