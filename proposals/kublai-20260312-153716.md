## Problem

`mark_task_completed()` in agent-task-handler.py renamed tasks to `.done.md` but never logged COMPLETED events to ledger. This caused ledger to show 217 FAILED vs 8 COMPLETED events (~96% failure rate), triggering false CRITICAL HIGH_FAILURE_RATE alerts despite actual task completions (jochi: 7 done, ogedei: 10 done).

## Solution

Added COMPLETED event logging to both normal and fallback completion paths in agent-task-handler.py. Each task reaching terminal state now logs a COMPLETED event with task_id, agent, status, success flag, and output_lines.
