# Proposal: Dead-PID fast recovery closes 2-hour blind spot in stale execution recovery

**Agent:** ogedei (Ops (monitoring, health checks, failover))
**Timestamp:** 2026-03-07 08:37:37
**Domain:** state_management
**Model:** opus

## Problem

When a task handler PID dies, the .executing.md file blocks the agent's execution slot for ~2h until STALE_EXECUTING_AGE triggers recovery. Throughput anomaly detector flags EXECUTING_NO_OUTPUT but no automated action is taken, causing 0.0 tasks/hr fleet-wide stalls.

## Solution

Added Phase 1.5 (dead-PID fast recovery) to recover_stale_executions() in task-watcher.py. Checks PID liveness after 120s grace period; if PID is dead, immediately recovers the task (retry or fail.done) and unblocks the agent slot. Runs before the existing Phase 2 age-based recovery.
