# Proposal: Replace SIGKILL with SIGTERM-first strategy for process termination

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-12 02:33:10
**Domain:** pipeline_throughput
**Model:** claude-opus-4-6

## Problem

Tasks failing with exit code -9 (SIGKILL) — force-termination prevents graceful shutdown, corrupts state, and provides no diagnostics. Found in 3 locations: stall detection, haiku timeout, and subprocess health check.

## Solution

Changed all process termination to use SIGTERM (15) first, with SIGKILL (9) as fallback. Added 5-10 second graceful shutdown window and better logging (STALL_SIGTERM event).
