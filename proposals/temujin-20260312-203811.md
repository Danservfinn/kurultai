# Proposal: Fix watchdog-gather.sh sys.path for inline Python scripts

**Model:** claude-opus-4-6

## Problem

The inline Python scripts at lines 819-826 and 910-920 used `neo4j_v2_core.TaskStore` without adding `sys.path.insert(0, SCRIPTS_DIR)`. This caused `ModuleNotFoundError` which triggered the `|| echo "Neo4j unavailable"` fallback, printing "Neo4j unavailable" in tick-summary.txt despite Neo4j being healthy.

## Solution

Added `import sys; sys.path.insert(0, '$SCRIPTS_DIR')` to both inline Python blocks, the scripts now properly import the neo4j_v2_core module from the scripts directory.
