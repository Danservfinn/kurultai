# Proposal: Add exclusive file lock to task-watcher.py to prevent duplicate daemon instances

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-07 00:33:55
**Domain:** task_dispatch
**Model:** opus

## Problem

Two task-watcher daemons running simultaneously (PIDs 75523, 75553), each with independent in-memory active_executions dict, causing 2 executing tasks per agent instead of 1 — doubling resource consumption, causing FileNotFoundError race conditions on task file renames, and contributing to 0% throughput

## Solution

Added fcntl.flock-based exclusive lock in task-watcher.py: new _acquire_daemon_lock() function with _DAEMON_LOCK_FILE, integrated into main() before daemon_loop() call with proper cleanup in finally block
