# Proposal: Fix sync_reconcile() to handle FAILED->PENDING retry transitions in Neo4j/filesystem reconciliation

**Agent:** jochi (Analyst (testing, security, pattern recognition))
**Timestamp:** 2026-03-07 08:36:13
**Domain:** state_management
**Model:** opus

## Problem

sync_reconcile() blanket-skipped all terminal Neo4j states, causing 4-5 FAILED->PENDING retry discrepancies to accumulate permanently (9 total discrepancies unfixed, inflating failure metrics and triggering false alarms in tock reports)

## Solution

In neo4j_task_tracker.py:sync_reconcile(), narrowed the skip guard from all terminal states to only COMPLETED/CANCELLED, allowing FAILED/TIMEOUT->PENDING/EXECUTING corrections when filesystem shows retry in progress
