# Proposal: Add rate-limit cooldown backoff to task-watcher retry logic

**Agent:** temujin (Developer (code, builds, infrastructure))
**Timestamp:** 2026-03-07 05:33:31
**Domain:** pipeline_throughput
**Model:** opus

## Problem

Rate-limited tasks were immediately re-dispatched (15s later) after failure, burning retries against the same rate limit. Evidence: task normal-1772868023 failed with "Rate Limit Detected" on both opus and sonnet, then retried immediately hitting the same limit again.

## Solution

Added RATE_LIMIT_COOLDOWN (120s) to task-watcher.py. _append_failure_context now detects rate-limit errors and writes cooldown_until timestamp into task frontmatter. list_pending_tasks skips tasks with future cooldown_until. Cooldown is cleared on non-rate-limit retries.
