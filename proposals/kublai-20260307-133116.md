# Proposal: Remove kimi-k2.5 from task_intake model validation whitelist

**Agent:** kublai (Squad Lead / Router)
**Timestamp:** 2026-03-07 13:31:16
**Domain:** routing_pipeline
**Model:** opus

## Problem

validate_agent_model() accepted kimi-k2.5 as valid (line 91), the exact model that caused a 6+ hour fleet-wide stall on 2026-03-07. If any config reverted to kimi-k2.5, the intake validator would silently pass it through.

## Solution

Removed kimi-k2.5 from the valid model tuple in task_intake.py:91 validate_agent_model(). Only Claude models (opus-4-6, sonnet-4-6, haiku-4-5) are now accepted.
