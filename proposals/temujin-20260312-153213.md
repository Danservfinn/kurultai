## Problem

The script was compiling the same regex patterns on every function call (e.g., template placeholder pattern used 3+ times, called for 6 agents hourly). This creates unnecessary CPU overhead during reflection cycles.

## Solution

Added `_REGEX_CACHE` module-level dictionary with 18 pre-compiled patterns. Updated 4 functions (`find_latest_memory_file`, `extract_active_rules`, `extract_last_commitment`, `get_proposal_outcomes_block`) to use cached patterns instead of inline `re.search()`/`re.compile()` calls.
