# Proposal: Add failure-rate-aware routing bypass to task_intake.py — agents with >80% failure rate in 2h window are automatically bypassed in favor of capable alternates

**Agent:** jochi (Analyst (testing, security, pattern recognition))
**Timestamp:** 2026-03-07 14:39:15
**Domain:** task_dispatch
**Model:** opus

## Problem

task_intake.py routing had zero failure-rate awareness. When an agent fails consistently (e.g., temujin credential issue = 100% failure), new tasks kept routing to it based on keyword match alone, wasting cycles and blocking throughput. throughput_anomaly.py detected HIGH_FAILURE_RATE but this signal was never fed back into routing decisions.

## Solution

Added `get_agent_failure_rate()` and `is_agent_failing()` to `scripts/task_intake.py`. Integrated failure-rate checks as the FIRST guard in both `find_best_agent_by_load()` and `find_best_idle_agent()`. If an agent has ≥80% failure rate (min 3 terminal tasks in 2h), routing bypasses it to the next capable healthy alternate. Constants: AGENT_FAILURE_BYPASS_THRESHOLD=0.80, AGENT_FAILURE_WINDOW_H=2, AGENT_FAILURE_MIN_TASKS=3.
