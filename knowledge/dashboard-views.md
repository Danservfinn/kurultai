# Dashboard Views Reference

**Application**: The Kurultai (`the-kurultai`)
**URL**: `https://the.kurult.ai` (also `http://127.0.0.1:18790`)
**Frontend**: Single-page application served from `index.html` with client-side routing

The dashboard has 7 tabs, each rendering a distinct view with data from the API.

---

## 1. Kanban

**Route**: `/` (default)
**Sub-views**: Tasks, Continuous Tasks

### Tasks Sub-view

Displays tasks organized into 4 kanban columns:

| Column | Neo4j Status | Description |
|--------|-------------|-------------|
| Pending | PENDING | Tasks waiting for execution |
| Executing | WORKING | Tasks currently being processed |
| Failed | FAILED | Tasks that failed execution |
| Done | COMPLETED | Successfully completed tasks (capped at 50) |

**API endpoints used**:
- `GET /api/tasks` -- load kanban columns
- `GET /api/agents/status` -- agent status badges (busy/available, pending/executing counts)
- `GET /api/tasks/:agent/:filename` -- task detail modal
- `GET /api/tasks/:agent/:filename/report` -- completion report modal
- `PUT /api/tasks/:agent/:filename` -- edit task body
- `POST /api/tasks/:agent/wake` -- wake idle agent
- `POST /api/tasks/:agent/:filename/retry` -- retry failed task
- `POST /api/tasks/:agent/retry-all` -- retry all failed for agent
- `POST /api/tasks/:agent/:filename/pause` -- pause pending task
- `POST /api/tasks/:agent/:filename/resume` -- resume paused task
- `POST /api/tasks/:agent/:filename/review` -- create review task
- `PUT /api/tasks/reorder` -- drag-and-drop reorder
- `POST /api/tasks/:agent/:filename/move-to-top` -- prioritize task
- `POST /api/queue/clear` -- redistribute overloaded queues

**Task cards show**: title, agent, priority badge, status, source, skill_hint, retry count, gate status.

### Continuous Tasks Sub-view

Shows long-running services and scheduled jobs:

| Type | Examples |
|------|----------|
| launchd | Hourly Reflection, Brainstorm Runner, Daily Backup |
| daemon | V2 Executor, Ogedei Watchdog, OpenClaw Gateway, Kurultai App Server, Cloudflare Tunnel |
| cron | Auto Dispatch, Log Rotation, Heartbeat Watchdog, Tock Gather |

Each entry shows: name, agent, schedule, status (healthy/warning/erroring/stopped/paused), last run time, next run time, consecutive successes/failures, PID.

**API endpoints used**:
- `GET /api/continuous-tasks` -- list all continuous tasks with health status
- `GET /api/continuous-tasks/logs?file=<path>` -- view log file tail in modal

## 2. Calendar

**Route**: `/calendar`

Displays calendar events from Signal calendar integration. Events can be filtered by agent (person).

**API endpoints used**:
- `GET /api/calendar/people` -- list agents for filter dropdown
- `GET /api/calendar/events?person=<name>` -- load events, optionally filtered

**Data source**: `~/.openclaw/apps/the-kurultai/data/calendar-events.json`

## 3. Reflections

**Route**: `/reflections`

Displays hourly reflection reports generated by the Kurultai system. Each reflection contains:

- **Executive Summary**: system status, key findings, infrastructure status
- **Fleet Status**: metrics table (tasks completed, success rate, agents active)
- **Agent Reflections**: per-agent blocks with grade (A-F), review score (/10), metrics (done/failed/velocity), worst moment, root cause, new rules, rule compliance, strengths/weaknesses
- **Grade Summary**: table of all agents with grades and key issues
- **Critical Alerts**: priority issues requiring attention
- **Cross-Agent Patterns**: systemic issues across agents
- **Actions Required**: prioritized action items with owners
- **New Rules Established**: WHEN/THEN behavioral rules from reflections
- **Bottom Line**: narrative assessment and action items

**Features**:
- Browse list of reflections (paginated, filterable by date/agent)
- Drill into individual reflection detail
- View per-agent reflection timeline
- View grade history chart per agent
- Audit trail: maps proposals to actual task files
- Rollback: create investigation task to undo reflection changes

**API endpoints used**:
- `GET /api/reflections` -- paginated list with summaries
- `GET /api/reflections/:id` -- full reflection detail
- `GET /api/reflections/:id/audit` -- audit trail
- `GET /api/agents/:name/reflections` -- agent-specific reflections
- `GET /api/per-agent-reflections/:id` -- standalone per-agent reflection
- `GET /api/agents/:name/grade-history` -- grade trend data
- `POST /api/rollback/:reflectionId` -- create rollback task
- `GET /api/proposals` -- aggregated proposals
- `GET /api/rules` -- aggregated rules
- `GET /api/hourly-reports` -- raw hourly report list
- `GET /api/hourly-reports/:filename` -- raw hourly report content

## 4. Wrappers

**Route**: `/wrappers`

Shows the claude-agent wrapper configuration and per-agent execution status:

- **Wrapper Script**: path, existence, last modified, fallback chain tiers
- **Per-Agent Cards**: configured model, base URL, token status, default match/drift, fallback chain, current task (from Neo4j WORKING status), recent task history
- **Process Status**: executing or idle

**API endpoints used**:
- `GET /api/agents/wrapper-status` -- comprehensive wrapper + agent status
- `GET /api/claude-agent/config` -- wrapper script configuration details

---

## 5. Sessions

**Route**: `/sessions`

Two sub-views:

### Active Sessions

Shows currently running `claude-agent` processes with:
- PID, agent name, settings type (primary/backup), model
- Current task (from Neo4j), duration, alive status
- Summary: total sessions, using primary vs backup

### Session History

Lists session JSONL files from each agent's `sessions/` directory with:
- Agent, status (active/drifted/stale/reset/backup), modification time, file size
- Session metadata: ID, start time, model/provider, event count, last event type
- Drill-down: view all events in a session
- Live stream: SSE endpoint for real-time event monitoring

**API endpoints used**:
- `GET /api/sessions/active` -- running processes
- `GET /api/sessions` -- session file list (filterable by agent, status)
- `GET /api/sessions/:agent/:filename` -- session detail with events
- `GET /api/sessions/:agent/:filename/stream` -- SSE live stream

## 6. Dispatch

**Route**: `/dispatch`

Shows task execution contexts from the unified task ledger (`task-ledger.jsonl`):

- Task lifecycle events: QUEUED -> EXECUTING -> COMPLETED/FAILED -> SCORED
- Per-task: task_id, agent, priority, summary, source, ACP prompt, executor
- Filterable by status (executing, queued, completed, failed)
- Drill-down: view all ledger events for a specific task

Also provides a trigger button to manually start a Kurultai reflection session.

**API endpoints used**:
- `GET /api/acp-contexts` (alias: `/api/dispatch-contexts`) -- task execution contexts
- `GET /api/acp-contexts/:taskId` -- specific task events
- `POST /api/sessions/trigger` -- trigger manual reflection
- `GET /api/dashboard` -- aggregated summary (tock, latest reflection, agent grades)
- `GET /api/tock/latest` -- system snapshot
- `GET /api/tock/history/:date` -- historical snapshots
- `GET /api/completions` -- task completion log
- `GET /api/verifications` -- verification log

## 7. Settings

**Route**: `/settings`

Three sections:

### Dispatch Mode

Switch between routing modes:
- **Auto**: Primary (Anthropic) first, backup (Z.AI) on failure
- **Backup**: Forced backup only (Z.AI/glm-5)
- **Primary**: Primary only, no fallback

### Model Selection

Change the primary Claude model:
- claude-opus-4-6
- claude-sonnet-4-6
- claude-haiku-4-5-20251001

### Provider Management

Manage LLM provider credentials:
- Anthropic: OAuth status (re-auth via `claude auth login`)
- Z.AI: token (masked), base URL, model
- Alibaba/DashScope: token (masked), base URL, model

Advanced model configuration:
- Per-agent model/base URL/token settings
- Bulk apply model to all agents
- Backup all config files
- View/edit raw config file content

**API endpoints used**:
- `GET /api/settings/mode` -- current dispatch mode
- `POST /api/settings/mode` -- update mode
- `GET /api/settings/model` -- current model
- `POST /api/settings/model` -- update model
- `GET /api/providers` -- provider configuration (protected)
- `POST /api/providers` -- update provider field (protected)
- `GET /api/settings/models` -- all model configs (protected)
- `PUT /api/settings/models/agent/:agent` -- update agent model (protected)
- `PUT /api/settings/models/all` -- bulk update all agents (protected)
- `POST /api/settings/models/backup` -- backup configs (protected)
- `GET /api/settings/models/files?path=` -- read config file (protected)
- `PUT /api/settings/models/files?path=` -- write config file (protected)
- `GET /api/settings/models/status` -- model health status (protected)
- `GET /api/agents/claude-config` -- per-agent profile/model/effort
