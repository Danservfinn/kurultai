## Prompt Weaknesses

1.  **Implicit Data Injection:** The prompt states "You will receive..." but does not provide a specific section or delimiter where that data should be pasted into the context window. This ambiguity likely led to the wrapper script failing to locate or inject the files correctly.
2.  **No Missing Data Protocol:** The prompt does not instruct the LLM on what to do if specific data sources (e.g., `reflections.txt`) are missing. A robust prompt should tell the LLM to flag missing data as a critical infrastructure failure rather than allowing the script to crash or the LLM to hallucinate.
3.  **Filesystem Dependency Illusion:** The prompt lists file paths implicitly (via the script error) but LLMs cannot access local filesystems. The prompt should clarify that data must be provided as text within the context, not referenced by path.
4.  **Lack of Pre-flight Check:** There is no instruction for the LLM to verify data integrity before beginning analysis. If data is partial, the LLM should warn the user immediately.
5.  **Context Window Management:** The prompt asks for 6 hours of data from 5 sources. Without token counting or summarization instructions, this could easily exceed context limits, leading to truncation or errors.

---

## Recommended Prompt Improvements

1.  **Explicit Data Injection Section:** Add a clearly marked section (e.g., `## INPUT DATA`) with delimiters where the wrapper script must paste the file contents.
2.  **Missing Data Handling:** Add instructions for the LLM to detect missing sections and report them as "Critical Infrastructure Failures" instead of hallucinating content.
3.  **Context Clarification:** Explicitly state that the LLM cannot access files directly and must rely solely on the text provided in the prompt.
4.  **Data Integrity Check:** Add a step for the LLM to confirm receipt of all 5 data sources before analyzing.
5.  **Error Recovery:** Instruct the LLM to provide a "Minimal Viable Analysis" if some data is missing, rather than failing silently.
