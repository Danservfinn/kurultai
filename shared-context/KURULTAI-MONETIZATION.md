# Kurultai Monetization Strategy

**Date:** 2026-03-04  
**Product:** Multi-Agent Orchestration Framework  
**Positioning:** "Kubernetes for AI Agents"

---

## Value Proposition

**What Kurultai Solves:**
- Single agents are limited; teams of agents accomplish more
- Multi-agent coordination is unsolved (shared state, task routing, conflict resolution)
- No production-ready orchestration framework exists
- Teams running 2+ agents need coordination infrastructure

**Our Moat:**
- 6-agent production system (Kublai, Temujin, Mongke, Chagatai, Jochi, Ogedei)
- Role-specific protocols (Developer, Researcher, Writer, Analyst, Ops, Lead)
- Hourly reflection + self-improvement engine
- Neo4j-backed shared state
- Task-watcher daemon (auto-dispatch)
- Battle-tested on our own operations

---

## Monetization Models

### Model 1: Open Core (RECOMMENDED)

**Free Tier:**
- 2 agents max
- Basic task routing
- Community support
- Self-hosted only

**Pro Tier ($99/mo):**
- Up to 10 agents
- Full reflection engine
- Neo4j shared state
- Priority support
- Analytics dashboard

**Enterprise Tier ($499/mo):**
- Unlimited agents
- Custom agent protocols
- SSO / RBAC
- Audit logs
- SLA support
- White-label option

**Why This Works:**
- Free tier drives adoption (developer love)
- Pro tier captures SMB/teams
- Enterprise tier for serious deployments
- Open source = community contributions

### Model 2: Usage-Based Pricing

**Pricing:**
- $0.10 per agent-hour
- $0.01 per task routed
- $0.001 per reflection cycle
- Free tier: 100 agent-hours/mo

**Why This Works:**
- Aligns cost with value
- Scales with customer success
- Low barrier to entry

**Risk:**
- Harder to predict revenue
- Customers may limit usage to save costs

---

### Model 3: Self-Hosted License

**Pricing:**
- Community: Free (GPL license)
- Commercial: $5,000/year (MIT license + support)
- Enterprise: $25,000/year (custom features + SLA)

**Why This Works:**
- Enterprises prefer self-hosted for security
- One-time revenue, low support overhead
- Clear upgrade path

**Risk:**
- Lower recurring revenue
- Piracy risk with open source

---

### Model 4: Managed Service (SaaS)

**Pricing:**
- Starter: $49/mo (3 agents, shared infra)
- Team: $199/mo (10 agents, dedicated infra)
- Enterprise: $999/mo (unlimited, VPC, SLA)

**Why This Works:**
- Highest margins (70-80%)
- Recurring revenue
- Full control over experience

**Risk:**
- Infrastructure costs
- Security/compliance burden
- Competition from cloud providers

---

### Model 5: Professional Services

**Offerings:**
- **Deployment:** $5,000 one-time (setup + hardening)
- **Custom Agents:** $10,000-50,000 (role-specific protocols)
- **Training:** $2,500/day (team workshops)
- **Managed Retainer:** $5,000/mo (monitoring + optimization)

**Why This Works:**
- Immediate revenue (no product build)
- Deep customer relationships
- Funds product development

**Risk:**
- Not scalable (trades time for money)
- Distracts from product focus

## Recommended Strategy: Hybrid Model

### Phase 1: Open Source + Services (Months 1-6)

**Goal:** Build adoption, fund development

**Actions:**
1. Open source Kurultai framework (MIT license)
2. Offer deployment services ($5K/deployment)
3. Build case studies with early adopters
4. Gather feedback, iterate on product

**Revenue Target:** $10K/mo (services)

---

### Phase 2: Managed SaaS Launch (Months 6-12)

**Goal:** Recurring revenue, product-market fit

**Actions:**
1. Launch Kurultai Cloud (managed service)
2. Free tier: 2 agents, 100 tasks/mo
3. Pro tier: $99/mo, 10 agents, unlimited tasks
4. Enterprise: Custom pricing

**Revenue Target:** $50K/mo (80% SaaS, 20% services)

---

### Phase 3: Enterprise + Ecosystem (Year 2+)

**Goal:** Scale, platform play

**Actions:**
1. Enterprise features (SSO, RBAC, audit logs)
2. Agent marketplace (third-party protocols)
3. Certification program (Kurultai partners)
4. Annual user conference

**Revenue Target:** $200K/mo (60% SaaS, 25% enterprise, 15% marketplace)
