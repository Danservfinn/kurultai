## Summary Header

Fixed critical frontend bug where the Kurultai reflections page was not displaying recent hourly reflection reports. The frontend only showed the 3:03 AM reflection (12+ hours old) despite 15 new reports existing on disk for today. Root causes included API parsing regex failures and frontend data model issues.

**Impact:** All 76 reflection reports now visible via API, 20 most recent returned by default with 5-minute auto-refresh.

---

## What Was Broken

### API Layer Issues

#### Parsing Regex Failure
The agent parsing regex required `Review Score: \d+\/\d+` format, but many reports had `Review Score: N/A`, causing parsing failures.

#### API Design Gap
The API was designed to return "proposals" extracted from reports, not the reports themselves. If a report had no proposals, it might not appear in the UI.

### Frontend Data Model Issues

#### Wrong Data Source
The frontend displayed proposals from `data.data.proposals` array, not the raw reports, causing incomplete display.

#### Missing Reports
Only 1 reflection showed (3:03 AM) when 15+ existed on disk for today.
