## Task Completion NULL Output Investigation

**Issue Identified:** Tasks are being marked as COMPLETED in Neo4j with NULL output and NULL result fields.

**Root Cause Analysis:**
- Location: `task-watcher.py` around line 1018
- Problem: `mark_task_completed()` is called with `execution_result.get("output")` and `execution_result.get("result")` which can be None
- Tasks get marked as COMPLETED even when `execution_result` doesn't contain output/result values

**Evidence:**
- Task `normal-1773851318-ed8a948f` ("Analyze Coca Cola testosterone study") shows status: COMPLETED, but output: NULL, result: NULL
- Task file doesn't exist in mongke's tasks directory
- Task exists in Neo4j with COMPLETED status

**Proposed Fix:**
Add validation before marking tasks as COMPLETED:
```python
# In task-watcher.py, around line 1018
if success and not execution_result.get("output"):
    success = False  # Don't mark as completed without output
    execution_result["output"] = "ERROR: Task completed without output"
```
